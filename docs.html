Getting Started with Extension Development

Plotune extensions allow you to extend the functionality of the core platform. Extensions can provide new data sources, visualizations, data processing capabilities, and more.

Prerequisites

  • Python 3.8+ installed
  • Basic understanding of REST APIs
  • Familiarity with WebSocket connections
  • Plotune Pro or Enterprise license

Extension Structure

Every Plotune extension must have the following structure:

your-extension/
├── __init__.py
├── main.py
├── manifest.json
├── requirements.txt
└── README.md

manifest.json

This file defines your extension's metadata:

{
    "name": "Your Extension",
    "id": "your_extension_id",
    "version": "1.0.0",
    "description": "Brief description of your extension",
    "mode": "lite|pro|enterprise",
    "author": "Your Name",
    "os": ["Windows", "Linux", "macOS"],
    "category": "DataSource|Visualization|Processing",
    "entry_point": "main:app"
}

REST API Reference v1.0.0

Plotune extensions communicate with the core platform through a REST API. Below are the essential endpoints every extension should implement.

GET
/config
Fetches the current configuration of the plugin and its status.
Parameters

No parameters required.

Response
Status: 200 OK
Content-Type: application/json
{
    "config": {
        // Current configuration parameters
    },
    "status": "running|paused|stopped"
}
Response Schema
config
object
A dictionary of all current configuration parameters
status
string
The status of the plugin (default: "paused")
GET
/stop
Sends a signal to stop the plugin. Note that this endpoint returns no body on success.
Response
Status: 200 OK
Content-Type: application/json
{}
GET
/status
Provides real-time status, uptime, and environment information for the plugin.
Response
Status: 200 OK
Content-Type: application/json
{
    "plugin": "Plugin Name",
    "status": "running",
    "init_time": "2023-08-15T12:30:45Z",
    "current_time": "2023-08-15T12:35:22Z",
    "uptime": "4 minutes 37 seconds",
    "environment": {
        "python": "3.10.6",
        "os": "Linux",
        "os_version": "5.15.0-78-generic",
        "host": "plotune-dev",
        "ip": "192.168.1.42"
    }
}
GET
/form
Retrieves the JSON schema for a user input form, allowing a client to dynamically build a UI.
Response
Status: 200 OK
Content-Type: application/json
{
    "title": "Data Source Configuration",
    "type": "object",
    "properties": {
        "source_name": {
            "type": "string",
            "title": "Source Name"
        },
        "sample_rate": {
            "type": "number",
            "title": "Sample Rate (Hz)",
            "default": 1000
        }
    },
    "required": ["source_name"]
}

WebSocket API v1.0.0

For real-time data streaming, Plotune uses WebSocket connections. This is essential for live data visualization.

WebSocket
/fetch/{signal_name}
A WebSocket endpoint that continuously streams data for a specified signal. The connection closes if an exception occurs during data fetching.
Parameters
Parameter Type Description Required
signal_name string Name of the signal to stream Yes
Message Format
{
    "timestamp": 1692100000.123456,
    "value": 42.5,
    "desc": "Temperature Sensor",
    "status": true
}
Message Schema
timestamp
float
Unix timestamp with microsecond precision
value
float
Current value of the signal
desc
string
Description of the signal
status
bool
Signal validity status (true = valid)
Example Implementation
@router.websocket("/fetch/{signal_name}")
async def websocket_endpoint(websocket: WebSocket, signal_name: str):
    """
    A WebSocket endpoint that continuously streams data for a specified signal.
    The connection closes if an exception occurs during data fetching.
    """
    await websocket.accept()
    try:
        while True:
            await asyncio.sleep(INTERVAL)
            payload = {
                "timestamp": time.time(),
                "value": get_sensor_value(signal_name),
                "desc": f"{signal_name}",
                "status": True
            }
            await websocket.send_json(payload)
    except Exception as e:
        await websocket.close()

Extension Development Best Practices

Performance Considerations

  • Optimize for low-latency data processing
  • Use asynchronous programming where possible
  • Limit the frequency of expensive operations
  • Implement proper error handling and recovery

Security Guidelines

  • Validate all input parameters
  • Sanitize data before processing
  • Use secure communication protocols
  • Implement proper authentication and authorization

User Experience

  • Provide clear error messages
  • Implement meaningful status indicators
  • Design intuitive configuration forms
  • Include comprehensive documentation
